# 10 — Performance & SEO Rules

> Fast on real devices and real networks = good UX. The site must feel instant on a mid-range Android phone.

## 1. Performance budget

- **Lighthouse (mobile) ≥ 90** for Performance, Accessibility, Best Practices, SEO.
- Ship **~zero client JS** by default (Astro). Only tiny vanilla islands where needed.
- Largest Contentful Paint target < 2.5s on mobile; CLS < 0.1; no long main-thread blocks.

## 2. Images & video

- Responsive `srcset` + WebP (auto-generated by `scripts/optimize-images.mjs`).
- Explicit `width`/`height` (or aspect-ratio) on all media → no layout shift.
- `loading="lazy"` for offscreen images; eager-load only the hero.
- Banner video: `preload="metadata"`, poster first, pause offscreen (see [03-video.md](03-video.md)).

## 3. Fonts

- Self-hosted, subset, only needed weights, `font-display: swap`, preload critical font(s). No render-blocking external font CSS.

## 4. CSS/JS

- Tailwind purges unused classes in production build.
- Inline critical CSS where Astro does so automatically (`inlineStylesheets: 'auto'`).
- Defer/lazy any non-critical script.

## 5. SEO

- Unique `<title>` + meta description per page (via layout props).
- Canonical URL, Open Graph + Twitter card meta, `og-image` (1200×630).
- Structured data (JSON-LD) for the travel business / trips where it adds value.
- `@astrojs/sitemap` generates the sitemap; keep `robots.txt` pointing at it.
- Descriptive, kebab-case URLs (`/trip/kashmir-great-lakes`).
- Semantic headings and real text content (don't bury copy in images).

## 6. Verification (Phase 5)

- Run Lighthouse on mobile + desktop.
- Test on throttled "Fast 3G" / mid-tier device emulation.
- Confirm no console errors, no 404 assets, sitemap valid.
